# talkdesk web（フロントエンドモック）

オンラインアシスタントとの業務コミュニケーションツール talkdesk の、**バックエンドなしで動作するフロントエンドモック**です。
Vite + React + TypeScript の SPA（CSR）で、`docs/frontend-spec.md` の画面仕様を実装しています。

## 起動方法

```bash
cd web
npm install
npm run dev      # http://localhost:5173 で起動
```

その他のスクリプト:

```bash
npm run build    # tsc（型チェック）+ vite build。警告なしで通ることを確認済み
npm run preview  # ビルド結果をローカルプレビュー
```

## デモユーザー

ログイン画面（`/login`）の「デモユーザーでログイン」から、パスワード入力なしでロールを切り替えて体験できます。

| デモユーザー | ロール | 体験できること |
|---|---|---|
| 鈴木 一郎 | 依頼者（メンバー） | 権限を付与されたチャンネルのみ閲覧。ステータスは「完了/差し戻し」のみ操作可 |
| 佐藤 花子 | クライアント管理者 | 全チャンネル閲覧、メンバー管理・チャンネル権限マトリクス |
| 田中 太郎 | アシスタント | 複数企業（サンプル社・ホゲホゲ社）を左端レールで切替。ステータスは「対応中/確認待ち」を操作可 |
| 運営 太一 | 運営管理者 | 企業発行・アシスタントのアサイン管理・読み取り専用チャンネルビュー |

任意のメール＋パスワードでも `api.login` は通ります（モックのため任意のパスワードで認証成功）。デモデータのメールアドレスは各ユーザーカードに表示されています。

デモ演出として、ログイン中は約40秒間隔で「相手からの新着メッセージ」が自動注入され、画面内トースト・未読バッジ・スレッド一覧の並び替えといったリアルタイム挙動を確認できます。

## アーキテクチャ（API層の差し替え方針）

実バックエンド接続時に**差し替える箇所を1枚に閉じ込める**設計です。

```
src/api/
  client.ts     TalkdeskApi インターフェース定義（REST/WebSocket相当の全操作）
  types.ts      ドメイン型（User / Channel / Thread / Message / Page ...）
  index.ts      ★ ここでモック実装を bind している（差し替えるのはこのファイル）
src/mocks/
  db.ts         in-memory のシードデータ（日本語のデモデータ）
  mockApi.ts    TalkdeskApi の in-memory 実装（権限チェック・enforce もここに実装）
  demo.ts       デモ用の新着メッセージ注入
```

- UI 層（`src/layout` / `src/pages` / `src/components`）は `src/api` のインターフェースにのみ依存し、**`src/mocks` を直接 import しない**規約です。
- リアルタイム更新は `api.subscribe()`（WebSocket相当のイベント）を `useApiData` フックで購読し、イベントごとに再取得して画面を最新化します。
- 実API接続時は、`TalkdeskApi` を満たす REST/WebSocket 実装を用意し、`src/api/index.ts` の `setApi(...)` を差し替えるだけで UI 層は無改変で動作します。
- 認可判定（チャンネル閲覧・スレッド／メッセージ取得・ステータス遷移・24時間以内の編集削除など）は UI の出し分けだけでなく `mockApi` 側でも enforce しており、実装時に API 層で強制すべき契約を明示しています。

## 既知の割り切り（モック固有の制約）

以下はモック（in-memory・単一タブ）ゆえの制約で、**実バックエンド接続で解消します**。

- **権限剥奪／無効化の即時反映は同一タブ内でのみ体験可能**：状態を各タブのメモリ上に持つため、タブ間の同期はありません（実装では WebSocket 経由で他クライアントにも即時反映される想定）。
- **添付ファイルのダウンロードは署名付きURL発行をトーストで表現**：ファイル名をクリックすると「署名付きURLを発行しました」というトーストを表示するのみで、実際のダウンロードは発火しません（実装では期限付き署名URLを都度発行して取得します）。
- **プロフィール変更後にフルリロードすると表示名が食い違う**：プロフィール（表示名・アバター）変更はモックDBに反映されますが、フルリロードするとモックDBがシードデータに戻る一方、ログイン状態は localStorage に永続化されるため、ヘッダーのアバターと各画面の表示名が一時的に食い違うことがあります（実装では永続DBに保存され不整合は起きません）。
- **通知は画面内トースト＋通知センターのみ**：メール／WebPush は送出しません。通知センターには「メンション・担当依頼のステータス変更・期日超過」のみを載せ、一般の新着メッセージはトースト＋未読バッジで通知します。
- **WebSocket 切断バナー**は UI を用意していますが、モックでは常時接続扱いのため表示されません。

## 技術構成

- Vite + React + TypeScript（SPA / CSR）
- React Router（data router / `useBlocker` による離脱ブロック対応）
- Zustand（認証・トースト等のクライアント状態）
- Tailwind CSS（UIコンポーネントライブラリ不使用・素組み）
- react-markdown + remark-gfm（ナレッジページの Markdown / GFM レンダリング）
