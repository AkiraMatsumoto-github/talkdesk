# talkdesk フロントエンド画面仕様

React SPA（CSR）の全画面のサイトマップ・ワイヤーフレーム・機能要件。
ワイヤーフレーム内の ①②③ は直下の機能要件の番号に対応する。
要件IDは画面プレフィックス（LOGIN-1 等）。requirements.md のFR/NFRに対応するものは末尾に併記。

## 0. サイトマップ（フロントの全ルート）

```
認証系（未ログインで到達可能）
├─ /login                          ログイン
├─ /password-reset                 パスワードリセット申請
└─ /invite/:token                  招待受諾（初回パスワード設定）

依頼者・クライアント管理者・アシスタント共通（要ログイン）
├─ /                               ホーム（最後に見たチャンネルへリダイレクト）
├─ /channels/:id                   チャンネル: スレッド一覧タブ
│   ├─ /channels/:id/threads/:tid    スレッド（右カラムに表示）
│   ├─ /channels/:id/files           ファイルタブ
│   └─ /channels/:id/pages           ナレッジタブ（ページ一覧）
│       ├─ /channels/:id/pages/:pid       ページ閲覧
│       ├─ /channels/:id/pages/:pid/edit  ページ編集
│       └─ /channels/:id/pages/:pid/history ページ編集履歴
├─ /settings/profile               プロフィール設定
└─ /settings/notifications         通知設定

クライアント管理者のみ
├─ /admin/members                  メンバー管理
└─ /admin/permissions              チャンネル権限マトリクス

アシスタントのみ（URL構造は共通ルートの前に /w/:orgId が付く）
└─ /w/:orgId/channels/:id/...      企業切り替えレール付きの同一画面

運営管理者のみ
├─ /ops/orgs                       企業一覧・発行
├─ /ops/orgs/:id                   企業詳細（アサイン管理）
└─ /ops/assistants                 アシスタント一覧・登録

モーダル（URLを持たない）
・チャンネル作成 ・スレッド作成 ・メンバー招待 ・ファイルアップロード確認
・ステータス変更確認 ・ページ編集競合の差分マージ
```

- ROUTE-1: 未ログインで保護ルートへアクセスしたら /login へ。ログイン後は元のURLへ復帰する
- ROUTE-2: 権限のないルート（他社チャンネル・管理者ページ等）は404相当の画面を出す（403で存在を教えない）
- ROUTE-3: `/` は最後に閲覧したチャンネルへリダイレクト（localStorageに保持）。初回は最初のチャンネル、チャンネルゼロなら空状態画面

## 1. 認証系

### 1.1 ログイン `/login`

```
┌──────────────────────────────────────┐
│              talkdesk                │
│   オンラインアシスタントとの窓口       │
│                                      │
│   メールアドレス                      │
│   ┌──────────────────────────────┐   │
│   │                            ① │   │
│   └──────────────────────────────┘   │
│   パスワード                          │
│   ┌──────────────────────────────┐   │
│   │                        [👁] ② │   │
│   └──────────────────────────────┘   │
│   ┌──────────────────────────────┐   │
│   │         ログイン           ③ │   │
│   └──────────────────────────────┘   │
│   パスワードをお忘れですか？ ④        │
│                                      │
│   ⚠ メールアドレスまたはパスワードが   │
│     正しくありません ⑤               │
└──────────────────────────────────────┘
```

- LOGIN-1: メール形式をクライアント側で検証。未入力時はログインボタン非活性
- LOGIN-2: パスワードは表示切替（👁）付き
- LOGIN-3: 送信中はボタンをスピナー化し二重送信を防ぐ。成功時はロールに応じて遷移（依頼者→ `/`、アシスタント→最後に見た企業の `/w/:orgId/`、運営→ `/ops/orgs`）（FR-A3）
- LOGIN-4: パスワードリセット申請ページへのリンク
- LOGIN-5: 認証失敗はメール存在の有無を区別しない同一メッセージ。5回失敗で一時ロック（Identity Platform側の設定を利用）
- LOGIN-6: 自己サインアップの導線は置かない（招待制。FR-A2）

### 1.2 パスワードリセット `/password-reset`

- RESET-1: メールアドレスを入力→「登録されている場合はメールを送信しました」と常に同文言で応答（アカウント存在を秘匿）
- RESET-2: メール内リンクから新パスワード設定（Identity Platformの標準フロー）

### 1.3 招待受諾 `/invite/:token`

```
┌──────────────────────────────────────┐
│  株式会社サンプル に招待されています ① │
│                                      │
│  メール: suzuki@example.com（変更不可）│
│  表示名   ┌────────────────────┐ ②   │
│  パスワード ┌────────────────────┐ ③  │
│  ┌──────────────────────────────┐    │
│  │       登録してはじめる        │    │
│  └──────────────────────────────┘    │
└──────────────────────────────────────┘
```

- INV-1: トークンを検証し、招待元企業名（アシスタント・運営招待の場合はtalkdesk運営）を表示。無効・期限切れ（7日）は専用エラー画面
- INV-2: 表示名は必須・50文字以内
- INV-3: パスワードは8文字以上・強度インジケータ表示。登録完了で自動ログインしホームへ

## 2. メインレイアウト（共通シェル）

依頼者は3カラム。アシスタントは左端に企業レールが付く（§6）。

```
┌────────────────────────────────────────────────────────────────────────┐
│ talkdesk │ 株式会社サンプル ①            ⚠切断中…再接続 ④  🔔③ ⚙ 😊② │
├─────────────┬──────────────────────┬───────────────────────────────────┤
│ (A)チャンネル │ (B)スレッド一覧       │ (C)スレッド                        │
│    一覧      │                      │                                   │
└─────────────┴──────────────────────┴───────────────────────────────────┘
```

- SHELL-1: ヘッダーに所属企業名（アシスタントは表示中の企業名）を常時表示
- SHELL-2: アバターメニュー: プロフィール設定 / 通知設定 / （管理者のみ）メンバー管理・権限管理 / ログアウト
- SHELL-3: 🔔通知センター: 自分宛メンション・担当依頼のステータス変更・期日超過を新着順に一覧。未読通知数をバッジ表示。クリックで該当スレッドへ
- SHELL-4: WebSocket切断時はヘッダー直下に黄色バナー「接続が切れました。再接続しています…」。再接続成功で自動的に消え、切断中の差分を取得して画面を最新化する（NFR-8）。5秒以内に指数バックオフで再接続
- SHELL-5: 画面幅1024px未満では(C)を全画面に切り替え、戻るナビゲーションを出す（タブレット縦向きの暫定対応。NFR-9）

## 3. チャンネルとスレッド

### 3.1 (A) チャンネル一覧カラム

```
┌─────────────────┐
│ チャンネル   [＋]①│
│ ─────────────── │
│ # 経費精算    ③ ②│
│ # 給与計算 ◀━━ ③│
│ # 請求書発行     │
│ # 備品購入    ①  │
│ ─────────────── │
│ ▽ アーカイブ済 ④ │
│ ─────────────── │
│ ⚙ メンバー管理 ⑤ │
└─────────────────┘
```

- CH-1: [＋]チャンネル作成はクライアント管理者・アシスタントのみ表示（FR-H9）。モーダルで名前・説明を入力。作成した管理者/アシスタント以外の依頼者(member)には権限付与されるまで見えない
- CH-2: 閲覧権限のあるチャンネルのみ表示（FR-H1）。未読メッセージ数バッジ＋未読あり太字。自分宛メンションがあるチャンネルは赤バッジ
- CH-3: 選択中チャンネルをハイライト。並びは名前順（並び替えは将来）
- CH-4: アーカイブ済みチャンネルは折りたたみ内に表示。読み取り専用（投稿・編集不可）
- CH-5: メンバー管理・権限管理への導線はクライアント管理者のみ表示
- CH-6: チャンネルゼロ（初期状態）の場合、(B)(C)に「アシスタントが業務チャンネルを作成するのをお待ちください」等の空状態を表示

### 3.2 (B) スレッド一覧カラム `/channels/:id`

```
┌────────────────────────────┐
│ # 給与計算  [スレッド|ファイル|ナレッジ] ① │
│ [＋ 新規スレッド] ②          │
│ [全て|依頼中|対応中|確認待ち|完了|ﾄﾋﾟｯｸ] ③│
│ ────────────────────────── │
│ ●確認待ち 🧵6月分給与計算  ③ ④│
│   期日7/15 田中:完了しました  │
│ ●対応中 ⚠期日超過           │
│   🧵住民税の更新手続き 7/1 ⑤ │
│ 💬交通費の質問               │
│   佐藤: 領収書は原本…        │
│ ────────────────────────── │
│ ▽ 完了済み (12) ⑥           │
└────────────────────────────┘
```

- TL-1: タブ切替（スレッド一覧/ファイル/ナレッジ）。URLに対応（/channels/:id, /files, /pages）
- TL-2: 新規スレッド作成モーダル。種別選択（🧵依頼 / 💬トピック）。依頼はタイトル・依頼内容・期日（任意）、トピックはタイトルのみ（FR-T1）
- TL-3: ステータスフィルタ（FR-H2）。既定は「全て（完了以外）」
- TL-4: 各行 = 種別アイコン・ステータス・タイトル・未読数・最終メッセージプレビュー・最終更新。最終更新の新しい順（FR-H2）
- TL-5: 期日超過の依頼スレッドに⚠バッジ（FR-T4）
- TL-6: 完了済みは折りたたみ（FR-T5）
- TL-7: 新着メッセージはWebSocketで行を即時更新し、並びを入れ替える

### 3.3 (C) スレッド表示 `/channels/:id/threads/:tid`

```
┌───────────────────────────────────────────────┐
│ 🧵 6月分給与計算                               │
│ ●確認待ち ① 期日:7/15 ②     [完了にする ▼] ③  │
│ 依頼内容: 6月分の給与計算をお願いします。        │
│ 対象は正社員12名＋アルバイト3名。 ④            │
│ ───────────────────────────────────────────── │
│ ┌ 田中(ｱｼｽﾀﾝﾄ) 7/10 14:02 ─────────────┐      │
│ │ 6月分の給与計算が完了しました。          │      │
│ │ 📎 給与一覧_2026-06.xlsx  1.2MB ⑤      │      │
│ └───────────────────────── 👁佐藤,鈴木 ⑥┘      │
│ ⚙ 田中さんがステータスを「確認待ち」に変更 ⑦     │
│ ─────────── ここから未読 ⑧ ───────────        │
│      ┌ 佐藤(自分) 15:30 ⋮ ⑨ ──────────┐       │
│      │ ありがとうございます。確認します。 │       │
│      └───────────────────────────────┘        │
│ ┌─────────────────────────────────────┐       │
│ │ @で誰かにメンション ⑩  📎 ⑪          │ [送信] │
│ └─────────────────────────────────────┘       │
└───────────────────────────────────────────────┘
```

- TH-1: ステータスバッジ。変更操作はロールで出し分け: アシスタントには「対応中にする」「確認待ちにする」、依頼者には「完了にする」「差し戻す（対応中へ）」のみ表示（FR-T2）。変更時は確認モーダル
- TH-2: 期日はインライン編集可（依頼者・アシスタント両方）。変更はシステムメッセージ化
- TH-3: ステータス変更ボタンは現在状態から遷移可能なもののみ活性
- TH-4: 依頼スレッドはヘッダーに依頼内容（body）を常時表示。長文は折りたたみ
- TH-5: 添付はファイル名・サイズ表示。クリックで署名付きURLを都度取得してダウンロード（FR-F2）。画像はインラインプレビュー
- TH-6: 既読者表示: メッセージ右下に既読ユーザーのアバターを表示（多数時は「+3」集約、ホバーで全員）（FR-H5）
- TH-7: システムメッセージ（ステータス・期日変更）は中央寄せの控えめ表示（FR-T3）
- TH-8: 「ここから未読」ライン。スレッドを最下部までスクロールしたら既読化しカーソルを送信（FR-H4）
- TH-9: 自分のメッセージの⋮メニュー: 編集・削除（投稿後24時間以内のみ活性。FR-H7）。削除確認後は「このメッセージは削除されました」表示。編集後は「(編集済み)」ラベル
- TH-10: @メンション: 入力補完はこのチャンネルを閲覧できるユーザーのみ（FR-H8）
- TH-11: 添付は複数可・1件100MB以内をクライアント側でも検査（FR-F3）。アップロード中はプログレスバー、失敗時はリトライボタン
- TH-12: 送信は楽観的更新（即座に表示→失敗時に「送信失敗・再試行」表示）（FR-H6）
- TH-13: メッセージはWebSocketで即時受信。過去分は上スクロールで50件ずつ遡り読み込み（NFR-4）
- TH-14: Enterで改行、Cmd/Ctrl+Enterで送信（誤爆防止。B2B利用を考慮）

## 4. ファイルタブ・ナレッジタブ

### 4.1 ファイルタブ `/channels/:id/files`

```
│ 🔍 ファイル名で検索 ①                          │
│ 📊 給与一覧_2026-06.xlsx  1.2MB 7/10 田中      │
│    └ 🧵6月分給与計算 ②            [⬇] ③       │
│ 📄 雇用契約書_山田.pdf  340KB 7/02 田中         │
```

- FILE-1: チャンネル配下全スレッドの添付を新しい順に一覧。ファイル名の部分一致検索（FR-F1）
- FILE-2: 元スレッドへのリンク（クリックでそのスレッド・該当メッセージ位置へ）
- FILE-3: ダウンロードは都度署名付きURL取得（FR-F2）。削除済みメッセージの添付は表示しない

### 4.2 ナレッジタブ `/channels/:id/pages`

```
│ ページ一覧                [＋ 新規ページ] ①      │
│ 📖 給与計算の手順          更新: 7/10 田中 ②    │
│ 📖 勤怠システムのログイン方法 更新: 6/28 佐藤     │

── ページ閲覧 /pages/:pid ──────────────────────
│ # 給与計算の手順        [編集] ③ [履歴] ④      │
│ 最終更新: 7/10 田中 (rev.8)                     │
│ （Markdownレンダリング表示） ⑤                  │

── ページ編集 /pages/:pid/edit ─────────────────
│ タイトル ┌─────────────────────┐               │
│ ┌──────────────┬──────────────┐               │
│ │ Markdown入力  │ プレビュー ⑥  │               │
│ └──────────────┴──────────────┘               │
│ 画像をドラッグで添付 ⑦        [保存] [破棄] ⑧   │
```

- KB-1: ページ作成・編集はチャンネル閲覧者全員可（FR-K2）
- KB-2: 一覧は更新の新しい順。タイトル検索
- KB-3: 編集画面は左右分割のライブプレビュー
- KB-4: 履歴画面: rev一覧（誰が・いつ）→ 過去版の閲覧・「この版に復元」（新revとして保存。FR-K3）
- KB-5: 本文はMarkdown（GFM: 表・チェックリスト対応）
- KB-6: 画像はドラッグ&ドロップで添付し、認可チェック付きAPI経由で配信（FR-K5）
- KB-7: 保存時にrevision不一致（他者が先に保存）なら競合モーダルで自分の変更と相手の変更の差分を提示し手動マージ（FR-K4）。編集中はローカルに自動下書き保存（リロード事故対策）

## 5. 設定・管理（クライアント）

### 5.1 プロフィール設定 `/settings/profile`

- SET-1: 表示名・アバター画像の変更。メールアドレスは表示のみ（変更は将来）
- SET-2: パスワード変更（現パスワード必須）

### 5.2 通知設定 `/settings/notifications`

```
│ 通知チャネル                                    │
│ [✓] メール通知 ①                               │
│ [✓] ブラウザ通知（WebPush） ②  [許可を再要求]   │
│ 未読をまとめて通知するまでの時間: [10分 ▼] ③     │
```

- NTF-1: メール通知のON/OFF
- NTF-2: WebPushのON/OFF。ブラウザ権限が未許可/拒否の場合は状態と再要求ボタンを表示。購読はデバイスごと
- NTF-3: ダイジェスト遅延（既読にならなかった未読をN分後にまとめて通知。既定10分）。メンションと担当依頼のステータス変更は即時通知
- NTF-4: Webview搭載時（デスクトップ/ネイティブ）はWebPush層をプラットフォーム通知に差し替える（プラットフォーム抽象化レイヤー経由）

### 5.3 メンバー管理 `/admin/members`（管理者のみ）

```
│ メンバー                       [＋ 招待] ①      │
│ 佐藤 (管理者)  sato@…    アクティブ             │
│ 鈴木          suzuki@…  アクティブ  [⋮] ②      │
│ 高橋          taka@…    招待中 ③   [再送][取消] │
```

- MEM-1: メール招待（複数まとめて可）。ロール選択（member/admin）（FR-P3）
- MEM-2: ⋮メニュー: 管理者昇格/降格・無効化。無効化は確認モーダル＋即時全セッション失効（FR-P3）
- MEM-3: 招待中メンバーは再送・取消が可能。招待の有効期限7日
- MEM-4: 操作はすべて監査ログに記録される旨を画面に明記（FR-P4）

### 5.4 チャンネル権限マトリクス `/admin/permissions`（管理者のみ）

```
│               佐藤(管理者)  鈴木   高橋          │
│ # 経費精算     ✓固定 ①     [✓]    [✓]          │
│ # 給与計算     ✓固定       [✓]    [ ] ②        │
│ # 請求書発行   ✓固定       [ ]    [✓]          │
│                          [変更を保存] ③        │
```

- PERM-1: 行=チャンネル、列=メンバー。管理者列は常に✓で編集不可（FR-P1）
- PERM-2: チェック操作はまとめて編集し、保存で一括反映。保存前の離脱は警告
- PERM-3: 保存時に剥奪されたユーザーの画面からは該当チャンネルが即時消える（WebSocket購読も解除。FR-P2）。付与されたユーザーには即時出現
- PERM-4: 変更内容は保存前に確認ダイアログで差分表示（「高橋さんから #給与計算 の閲覧権限を外します」）

## 6. アシスタント用ビュー `/w/:orgId/...`

```
┌──┬─────────────────────────────────────────┐
│🅰 │ （§2〜4と同一の3カラム画面）              │
│──│                                          │
│🅱③│                                          │
│🅲 │                                          │
│⚠ │                                          │
└──┴─────────────────────────────────────────┘
```

- RAIL-1: 左端レールにアサイン企業のアイコン（最大10社）。企業ごとの未読合計バッジ、メンションは赤（FR-S1）
- RAIL-2: クリックで `/w/:orgId/` へ切替。チャンネル・スレッド状態は企業ごとに独立（FR-S2）
- RAIL-3: 表示中でない企業の新着もレールのバッジにリアルタイム反映。通知は全担当企業分受信（FR-S3）
- RAIL-4: 依頼者と同一コンポーネントを使い、レールの有無だけで分岐する（実装方針）

## 7. 運営管理画面 `/ops`

### 7.1 企業一覧 `/ops/orgs`

- OPS-1: 企業の一覧（名前・メンバー数・チャンネル数・担当アシスタント）と新規発行（FR-O1: 企業名＋初期管理者メールを入力→招待送信）

### 7.2 企業詳細 `/ops/orgs/:id`

- OPS-2: 担当アシスタントのアサイン/解除。アサイン時に上限10社超過ならエラー表示（FR-O2）
- OPS-3: 企業のチャンネル一覧を閲覧できる（クリックで読み取り専用ビュー）。閲覧した事実が監査ログに残る旨を画面上部に常時表示（FR-O3）

### 7.3 アシスタント一覧 `/ops/assistants`

- OPS-4: アシスタントの招待・無効化・担当企業数の一覧（n/10表示）

## 8. 通知の画面挙動（横断）

- NOTIF-1: アプリ表示中の新着は画面内トースト＋通知センター追加のみ（WebPushは出さない）
- NOTIF-2: 非表示（タブ非アクティブ・アプリ未起動）の場合にWebPush。クリックで該当スレッドを直接開く
- NOTIF-3: WebPush本文にメッセージ内容は含めない（「#給与計算 に新着があります」形式。共有PCでの覗き見対策）
- NOTIF-4: メール通知も同様に本文非掲載＋リンクのみ

## 9. 共通状態・エラー処理

- STATE-1: 一覧系のローディングはスケルトン表示。空状態は用途別の案内文＋主要アクション（例: スレッドゼロ→「最初の依頼を作成しましょう [＋新規スレッド]」）
- STATE-2: APIエラーはトースト（再試行可能なものは再試行ボタン付き）。権限エラー(403相当)は「アクセスできません」画面へ
- STATE-3: 楽観的更新の失敗はロールバックして失敗表示（送信メッセージは「再試行/破棄」の選択を残す）
- STATE-4: 認証トークン失効時は現在URLを保持して /login へ。再ログインで復帰
