# talkdesk サイトマップ・ワイヤーフレーム・要件定義

情報構造は **チャンネル（業務・権限単位） > スレッド（依頼/トピック） > メッセージ** の3層。

## 1. サイトマップ

```
/login                          ログイン
/invite/:token                  招待受諾（初回パスワード設定）

【依頼者・クライアント管理者】
/                               ホーム（チャンネル一覧 + スレッド一覧 + スレッド）
/channels/:id                   チャンネル（スレッド一覧）
/channels/:id/threads/:tid      スレッド（メッセージ）
/channels/:id/files             チャンネル（ファイルタブ）
/channels/:id/pages             チャンネル（ナレッジタブ: ページ一覧）
/channels/:id/pages/:pid        ナレッジページ閲覧
/channels/:id/pages/:pid/edit   ナレッジページ編集
/settings/profile               個人設定（プロフィール・通知設定）
/admin/members                  メンバー管理（管理者のみ）
/admin/permissions              チャンネル権限マトリクス（管理者のみ）

【アシスタント】
/w/:orgId/...                   上記と同じ画面構成に企業切り替えレールが付く

【運営管理者】
/ops/orgs                       企業一覧・発行
/ops/orgs/:id                   企業詳細（アシスタントのアサイン管理）
/ops/assistants                 アシスタント一覧・登録
```

## 2. ワイヤーフレームと機能要件

### 2.1 ログイン `/login`

```
┌──────────────────────────────────────┐
│                                      │
│              talkdesk                │
│                                      │
│   ┌──────────────────────────────┐   │
│   │ メールアドレス                │   │
│   └──────────────────────────────┘   │
│   ┌──────────────────────────────┐   │
│   │ パスワード                    │   │
│   └──────────────────────────────┘   │
│   [ ログイン ]                        │
│   パスワードをお忘れですか？           │
│                                      │
└──────────────────────────────────────┘
```

**機能要件**
- FR-A1: メール+パスワード認証（Identity Platform）。パスワードリセットあり
- FR-A2: ユーザー登録は招待制のみ（自己サインアップなし）。招待メールのリンクから初回パスワード設定
- FR-A3: ログイン後、ロールに応じてホーム（依頼者/アシスタント）または /ops（運営）へ遷移

### 2.2 ホーム＝3カラム（依頼者）

左からチャンネル一覧 → 選択チャンネルのスレッド一覧 → 選択スレッドのメッセージ。

```
┌──────────────────────────────────────────────────────────────────────────┐
│ talkdesk │ 株式会社サンプル                              🔔 ⚙ (アバター)  │
├─────────────┬──────────────────────┬───────────────────────────────────────┤
│ チャンネル   │ # 給与計算           │ 🧵 6月分給与計算                       │
│         [＋]│ [＋ 新規スレッド]     │ ● 確認待ち   期日: 7/15   [完了にする] │
│             │ ────────────────────│───────────────────────────────────────│
│ # 経費精算 ③│ フィルタ:            │ ┌ 田中(アシスタント) 7/10 14:02 ────┐ │
│ # 給与計算◀━│ [全て|依頼|ﾄﾋﾟｯｸ|完了]│ │ 6月分の給与計算が完了しました。     │ │
│ # 請求書発行│ ────────────────────│ │ ご確認ください。                    │ │
│ # 備品購入 ①│ ●確認待ち           │ │ 📎 給与一覧_2026-06.xlsx           │ │
│             │ 🧵6月分給与計算    ③│ └──────────────────────── 👁佐藤,鈴木 ┘ │
│             │   田中: 完了しました │      ┌ 佐藤(自分) 7/10 15:30 ───────┐ │
│             │ ●対応中             │      │ ありがとうございます。確認します│ │
│             │ 🧵住民税の更新手続き │      └───────────────────────────────┘ │
│             │ 💬交通費の質問       │ ─────────── ここから未読 ───────────   │
│             │ ○完了               │                                       │
│⚙ メンバー管理│ 🧵5月分給与計算      │───────────────────────────────────────│
│ (管理者のみ) │                     │ ┌───────────────────────┐ [📎] [送信] │
│             │                     │ │ メッセージを入力…      │             │
└─────────────┴──────────────────────┴─┴───────────────────────┴─────────────┘
```

**機能要件**
- FR-H1: 左カラムに閲覧権限のあるチャンネルのみ表示。未読件数バッジ、未読ありチャンネルを太字表示
- FR-H2: 中カラムは選択チャンネルのスレッド一覧。種別（🧵依頼 / 💬トピック）・ステータス・最終メッセージプレビュー・未読数を表示し、最終更新の新しい順に並べる。ステータスでフィルタできる
- FR-H3: メッセージはWebSocketでリアルタイム受信。切断時は自動再接続し、最終受信ID以降を差分取得
- FR-H4: スレッド内に「ここから未読」ラインを表示。スレッドを最下部まで見たら既読化
- FR-H5: 既読者の表示（Slack風）。メッセージごとに「誰が読んだか」をアイコンで表示する。給与情報などを確認したことの証跡としても機能する
- FR-H6: メッセージ投稿（テキスト＋ファイル添付、複数可）。送信は楽観的更新
- FR-H7: メッセージの編集・削除は自分の投稿のみ、投稿後24時間以内。編集済みラベルを表示。削除時は本文・添付を消し「このメッセージは削除されました」の痕跡を残す（完全な非表示にはしない）
- FR-H8: @メンション（チャンネル閲覧者のみ候補に出す）。メンションされたら通知を優先度高で送る
- FR-H9: チャンネル作成 [＋] は管理者・アシスタントのみ表示（権限マトリクス準拠）

### 2.3 スレッド（依頼スレッド）

**機能要件**
- FR-T1: スレッドは2種類。**依頼スレッド**（タイトル・依頼内容・期日・ステータス）と**トピックスレッド**（タイトルのみ）。スレッド作成は閲覧可能なチャンネルの全員ができる
- FR-T2: ステータスは 依頼中→対応中→確認待ち→完了。アシスタントは「依頼中→対応中→確認待ち」、依頼者は「確認待ち→完了」承認と差し戻し（→対応中）
- FR-T3: ステータス変更・期日変更はスレッド内にシステムメッセージとして自動投稿する（変更履歴がやり取りの中で辿れる）
- FR-T4: 期日超過の依頼スレッドはバッジ表示＋前日と当日にリマインド通知
- FR-T5: 完了済みスレッドは既定でスレッド一覧の下部に畳む（フィルタで表示可）

### 2.4 チャンネルのファイルタブ `/channels/:id/files`

```
├─[スレッド一覧]──[ファイル]──[ナレッジ]─────────────────────────────│
│ 🔍 ファイル名で検索                                                  │
│ ┌──────────────────────────────────────────────────────┐           │
│ │ 📊 給与一覧_2026-06.xlsx   1.2MB  7/10  田中           │           │
│ │    └ スレッド: 6月分給与計算                [⬇ DL]    │           │
│ │ 📄 雇用契約書_山田.pdf      340KB  7/02  田中           │           │
│ └──────────────────────────────────────────────────────┘           │
```

**機能要件**
- FR-F1: チャンネル配下の全スレッドの添付を時系列で一覧。投稿者・元スレッドへのリンク付き
- FR-F2: ダウンロードは署名付きURL（有効期限あり）。権限チェックはURL発行時にAPI側で実施
- FR-F3: 1ファイル上限100MB、保存期間は無制限。ウイルススキャンは将来検討

### 2.4b チャンネルのナレッジタブ `/channels/:id/pages`（業務マニュアル）

```
├─[スレッド一覧]──[ファイル]──[ナレッジ]─────────────────────────────│
│ ページ一覧                                    [＋ 新規ページ]        │
│ ┌──────────────────────────────────────────────────────┐           │
│ │ 📖 給与計算の手順            更新: 7/10 田中           │           │
│ │ 📖 勤怠システムのログイン方法  更新: 6/28 佐藤           │           │
│ └──────────────────────────────────────────────────────┘           │
│                                                                    │
│ ── ページ閲覧 ─────────────────────────────────────────             │
│ # 給与計算の手順                        [編集] [履歴]                │
│ 最終更新: 7/10 田中                                                 │
│                                                                    │
│ 1. 毎月25日までに勤怠データをエクスポート                             │
│ 2. 控除項目を確認（住民税は6月に更新）                                │
│ 3. 計算結果を依頼スレッドに添付して報告                               │
```

**機能要件**
- FR-K1: チャンネルごとに複数のナレッジページを作成できる。スレッド（フロー）とは独立したストック情報
- FR-K2: 本文はMarkdown。編集はチャンネル閲覧者なら誰でも可（権限マトリクス準拠）
- FR-K3: 保存ごとに編集履歴（誰が・いつ・全文スナップショット）を保持し、過去版を閲覧・復元できる
- FR-K4: 同時編集は後勝ちにせず、保存時に他者更新を検知したら差分を提示して手動マージ（楽観ロック）
- FR-K5: ページ内画像は添付の仕組みを流用し、表示時に認可チェック付きで配信する（署名付きURLの直埋め込みはしない）

### 2.5 チャンネル権限マトリクス `/admin/permissions`（クライアント管理者）

```
┌────────────────────────────────────────────────────────┐
│ チャンネル閲覧権限                                       │
│                                                        │
│              │ 佐藤(管理者) │ 鈴木  │ 高橋  │           │
│ ─────────────┼─────────────┼──────┼──────┤           │
│ # 経費精算    │ ✓(固定)     │ ✓    │ ✓    │           │
│ # 給与計算    │ ✓(固定)     │ ✓    │ ✕    │           │
│ # 請求書発行  │ ✓(固定)     │ ✕    │ ✓    │           │
│                                        [保存]          │
└────────────────────────────────────────────────────────┘
```

**機能要件**
- FR-P1: 管理者は自社メンバー×チャンネルの閲覧権限をマトリクスで一括編集できる。管理者自身は常に全チャンネル閲覧可（固定）
- FR-P2: 権限を剥奪されたユーザーは即時にそのチャンネルが見えなくなる（接続中のWebSocket購読も解除）
- FR-P3: メンバー招待（メール）、無効化（退職時）。無効化で全セッション失効
- FR-P4: 権限変更は監査ログに記録（誰が・いつ・誰に・何を）

### 2.6 アシスタント用ビュー（企業切り替えレール付き）

```
┌──┬───────────────────────────────────────────────────────────────────┐
│🅰 │ talkdesk │ 株式会社サンプル(A社)                🔔 ⚙ (アバター)    │
│──├─────────────┬──────────────────┬──────────────────────────────────┤
│🅱③│ チャンネル   │ # 給与計算        │ 🧵 6月分給与計算                  │
│  │             │ （スレッド一覧）   │ （以下、依頼者と同じ3カラム構成）  │
│🅲 │ # 経費精算 ①│                  │                                  │
│  │ # 給与計算◀━│                  │                                  │
│＋│             │                  │                                  │
└──┴─────────────┴──────────────────┴──────────────────────────────────┘
 ↑ 左端レール: 担当企業(最大10社)。未読バッジ付き。クリックで企業切替
```

**機能要件**
- FR-S1: レールにはアサインされた企業のみ表示。企業単位の未読合計バッジ
- FR-S2: 企業を跨いだ状態は持たない（表示中の企業のチャンネルだけ購読・表示）。切替時にチャンネル一覧・スレッドが丸ごと入れ替わる
- FR-S3: 通知は全担当企業分を受け取る（表示中でない企業も）

### 2.7 運営管理画面 `/ops`

```
┌────────────────────────────────────────────────────────┐
│ talkdesk ops   [企業] [アシスタント]                     │
├────────────────────────────────────────────────────────┤
│ 企業一覧                              [＋ 企業を発行]    │
│ ┌────────────────────────────────────────────────┐     │
│ │ 株式会社サンプル   メンバー3  チャンネル4         │     │
│ │   担当: 田中, 伊藤                [アサイン管理]  │     │
│ │ 株式会社ホゲホゲ   メンバー5  チャンネル7         │     │
│ └────────────────────────────────────────────────┘     │
└────────────────────────────────────────────────────────┘
```

**機能要件**
- FR-O1: 企業アカウント発行（企業名＋初期管理者のメール招待）
- FR-O2: アシスタント登録と企業へのアサイン/解除（1人最大10社のバリデーション）
- FR-O3: 運営管理者は全企業のチャンネルを閲覧できる（サポート・品質管理目的。閲覧は監査ログに記録）

## 3. 非機能要件

| # | 分類 | 要件 |
|---|---|---|
| NFR-1 | セキュリティ | 全認可判定（チャンネル一覧・スレッド・メッセージ・ファイルURL発行・WebSocket購読）をAPI層で強制。organization_id境界の自動テストを必須にする |
| NFR-2 | セキュリティ | 通信はTLSのみ。添付ファイルは非公開バケット＋署名付きURL（有効期限10分） |
| NFR-3 | セキュリティ | 監査ログ: 権限変更・メンバー招待/無効化・運営の企業データ閲覧を記録し、削除不可のストレージに保管 |
| NFR-4 | 性能 | メッセージ送信→他クライアント表示まで1秒以内（p95）。スレッド初期表示は直近50件をページング |
| NFR-5 | 性能 | 同時接続: 初期は 100社 × 平均5接続 = 500 WebSocket接続を1〜2インスタンスで捌ける設計（Goなら余裕） |
| NFR-6 | 可用性 | 営業時間帯（平日9-18時）の稼働率99.5%目標。Cloud Run min-instances=1（コールドスタート回避） |
| NFR-7 | データ保護 | Cloud SQL自動バックアップ（日次・7日保持）＋ポイントインタイムリカバリ。GCSはバージョニング有効 |
| NFR-8 | 切断耐性 | WebSocket切断（Cloud Run 60分制限・ネットワーク断・スリープ復帰）から5秒以内に自動再接続し、欠落メッセージを差分取得 |
| NFR-9 | 対応環境 | ブラウザ: Chrome/Edge/Safari 最新2バージョン。画面幅1024px以上を主対象（モバイルはネイティブアプリで対応予定） |
| NFR-10 | 言語 | 日本語のみ（i18n基盤は入れない） |
