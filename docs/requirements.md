# talkdesk サイトマップ・ワイヤーフレーム・要件定義

## 1. サイトマップ

```
/login                        ログイン
/invite/:token                招待受諾（初回パスワード設定）

【依頼者・クライアント管理者】
/                             ホーム（スレッド一覧 + チャット）
/threads/:id                  スレッド（チャットタブ）
/threads/:id/requests         スレッド（依頼一覧タブ）
/threads/:id/requests/:rid    依頼カード詳細
/threads/:id/files            スレッド（ファイルタブ）
/settings/profile             個人設定（プロフィール・通知設定）
/admin/members                メンバー管理（管理者のみ）
/admin/permissions            スレッド権限マトリクス（管理者のみ）

【アシスタント】
/w/:orgId/...                 上記と同じ画面構成に企業切り替えレールが付く

【運営管理者】
/ops/orgs                     企業一覧・発行
/ops/orgs/:id                 企業詳細（アシスタントのアサイン管理）
/ops/assistants               アシスタント一覧・登録
```

## 2. ワイヤーフレームと機能要件

### 2.1 ログイン `/login`

```
┌──────────────────────────────────────┐
│                                      │
│              talkdesk                │
│                                      │
│   ┌──────────────────────────────┐   │
│   │ メールアドレス                │   │
│   └──────────────────────────────┘   │
│   ┌──────────────────────────────┐   │
│   │ パスワード                    │   │
│   └──────────────────────────────┘   │
│   [ ログイン ]                        │
│   パスワードをお忘れですか？           │
│                                      │
└──────────────────────────────────────┘
```

**機能要件**
- FR-A1: メール+パスワード認証（Identity Platform）。パスワードリセットあり
- FR-A2: ユーザー登録は招待制のみ（自己サインアップなし）。招待メールのリンクから初回パスワード設定
- FR-A3: ログイン後、ロールに応じてホーム（依頼者/アシスタント）または /ops（運営）へ遷移

### 2.2 ホーム＝メイン3ペイン（依頼者）

```
┌────────────────────────────────────────────────────────────────────┐
│ talkdesk │ 株式会社サンプル                          🔔 ⚙ (アバター)│
├──────────────────┬─────────────────────────────────────────────────┤
│ スレッド    [＋]  │ # 給与計算              メンバー(4)  ⚙          │
│                  ├─[チャット]──[依頼一覧]──[ファイル]───────────────│
│ # 経費精算    ③  │                                                 │
│ # 給与計算  ◀━━  │  ┌ 田中(アシスタント) 7/10 14:02 ─────────┐      │
│ # 請求書発行      │  │ 6月分の給与計算が完了しました。          │      │
│ # 備品購入    ①  │  │ ご確認ください。                        │      │
│                  │  │ 📎 給与一覧_2026-06.xlsx                │      │
│                  │  └────────────────────────────────────────┘      │
│                  │        ┌─ 佐藤(自分) 7/10 15:30 ──────────┐      │
│                  │        │ ありがとうございます。確認します。 │      │
│                  │        └───────────────────────────────────┘      │
│                  │  ─────────── ここから未読 ───────────              │
│                  ├─────────────────────────────────────────────────┤
│ ⚙ メンバー管理    │ ┌─────────────────────────────────┐ [📎] [依頼] │
│   (管理者のみ)    │ │ メッセージを入力…                │  [送信]     │
└──────────────────┴─┴─────────────────────────────────┴─────────────┘
```

**機能要件**
- FR-H1: 左ペインに閲覧権限のあるスレッドのみ表示。未読件数バッジ、未読ありスレッドを太字表示
- FR-H2: メッセージはWebSocketでリアルタイム受信。切断時は自動再接続し、最終受信ID以降を差分取得
- FR-H3: 「ここから未読」ラインを表示。スレッドを開いて最下部まで見たら既読化
- FR-H3b: 既読者の表示（Slack風）。メッセージごとに「誰が読んだか」をアイコンで表示する。給与情報などを確認したことの証跡としても機能する
- FR-H4: メッセージ投稿（テキスト＋ファイル添付、複数可）。送信は楽観的更新
- FR-H5: メッセージの編集・削除は自分の投稿のみ、投稿後24時間以内。編集済みラベルを表示。削除時は本文・添付を消し「このメッセージは削除されました」の痕跡を残す（完全な非表示にはしない）
- FR-H6: @メンション（スレッド参加者のみ候補に出す）。メンションされたら通知を優先度高で送る
- FR-H7: スレッド作成 [＋] は管理者・アシスタントのみ表示（FR権限マトリクス準拠）

### 2.3 依頼一覧タブ `/threads/:id/requests`

```
├─[チャット]──[依頼一覧]──[ファイル]──────────────────────────────────│
│                                            [＋ 新規依頼]            │
│ ┌──────────────────────────────────────────────────────┐           │
│ │ ● 確認待ち  6月分給与計算          期日: 7/15         │           │
│ │   最終更新: 7/10 田中「計算完了しました」📎1           │           │
│ ├──────────────────────────────────────────────────────┤           │
│ │ ● 対応中    住民税の更新手続き      期日: 7/20         │           │
│ ├──────────────────────────────────────────────────────┤           │
│ │ ○ 完了      5月分給与計算          完了日: 6/13       │           │
│ └──────────────────────────────────────────────────────┘           │
│ フィルタ: [すべて|依頼中|対応中|確認待ち|完了]                        │
```

**機能要件**
- FR-R1: 依頼カード＝タイトル・詳細・期日・ステータス・添付。ステータスは 依頼中→対応中→確認待ち→完了
- FR-R2: 遷移権限: アシスタントは「依頼中→対応中→確認待ち」、依頼者は「確認待ち→完了」承認と差し戻し（→対応中）
- FR-R3: カードにはチャットと同UIのコメント欄が付き、依頼に紐づくやり取り・成果物を集約
- FR-R4: カードの新着コメント・ステータス変更はスレッドのチャットにもシステムメッセージとして流す（動線の一本化）
- FR-R5: 期日超過の依頼はバッジ表示＋前日と当日にリマインド通知

### 2.4 ファイルタブ `/threads/:id/files`

```
├─[チャット]──[依頼一覧]──[ファイル]──────────────────────────────────│
│ 🔍 ファイル名で検索                                                  │
│ ┌──────────────────────────────────────────────────────┐           │
│ │ 📊 給与一覧_2026-06.xlsx   1.2MB  7/10  田中           │           │
│ │    └ 依頼: 6月分給与計算                    [⬇ DL]    │           │
│ │ 📄 雇用契約書_山田.pdf      340KB  7/02  田中           │           │
│ └──────────────────────────────────────────────────────┘           │
```

**機能要件**
- FR-F1: スレッド内の全添付を時系列で一覧。投稿者・紐づく依頼カードへのリンク付き
- FR-F2: ダウンロードは署名付きURL（有効期限あり）。権限チェックはURL発行時にAPI側で実施
- FR-F3: 1ファイル上限100MB、保存期間は無制限。ウイルススキャンは将来検討

### 2.5 スレッド権限マトリクス `/admin/permissions`（クライアント管理者）

```
┌────────────────────────────────────────────────────────┐
│ スレッド閲覧権限                                         │
│                                                        │
│              │ 佐藤(管理者) │ 鈴木  │ 高橋  │           │
│ ─────────────┼─────────────┼──────┼──────┤           │
│ # 経費精算    │ ✓(固定)     │ ✓    │ ✓    │           │
│ # 給与計算    │ ✓(固定)     │ ✓    │ ✕    │           │
│ # 請求書発行  │ ✓(固定)     │ ✕    │ ✓    │           │
│                                        [保存]          │
└────────────────────────────────────────────────────────┘
```

**機能要件**
- FR-P1: 管理者は自社メンバー×スレッドの閲覧権限をマトリクスで一括編集できる。管理者自身は常に全スレッド閲覧可（固定）
- FR-P2: 権限を剥奪されたユーザーは即時にそのスレッドが見えなくなる（接続中のWebSocket購読も解除）
- FR-P3: メンバー招待（メール）、無効化（退職時）。無効化で全セッション失効
- FR-P4: 権限変更は監査ログに記録（誰が・いつ・誰に・何を）

### 2.6 アシスタント用ビュー（企業切り替えレール付き）

```
┌──┬─────────────────────────────────────────────────────────────────┐
│🅰 │ talkdesk │ 株式会社サンプル(A社)              🔔 ⚙ (アバター)   │
│──├──────────────────┬──────────────────────────────────────────────┤
│🅱③│ スレッド    [＋]  │ # 給与計算                                   │
│  │                  │  （以下、依頼者と同じ3ペイン構成）              │
│🅲 │ # 経費精算    ①  │                                              │
│  │ # 給与計算  ◀━━  │                                              │
│＋│                  │                                              │
└──┴──────────────────┴──────────────────────────────────────────────┘
 ↑ 左端レール: 担当企業(最大10社)。未読バッジ付き。クリックで企業切替
```

**機能要件**
- FR-S1: レールにはアサインされた企業のみ表示。企業単位の未読合計バッジ
- FR-S2: 企業を跨いだ状態は持たない（表示中の企業のスレッドだけ購読・表示）。切替時にスレッド一覧・チャットが丸ごと入れ替わる
- FR-S3: 通知は全担当企業分を受け取る（表示中でない企業も）

### 2.7 運営管理画面 `/ops`

```
┌────────────────────────────────────────────────────────┐
│ talkdesk ops   [企業] [アシスタント]                     │
├────────────────────────────────────────────────────────┤
│ 企業一覧                              [＋ 企業を発行]    │
│ ┌────────────────────────────────────────────────┐     │
│ │ 株式会社サンプル   メンバー3  スレッド4           │     │
│ │   担当: 田中, 伊藤                [アサイン管理]  │     │
│ │ 株式会社ホゲホゲ   メンバー5  スレッド7           │     │
│ └────────────────────────────────────────────────┘     │
└────────────────────────────────────────────────────────┘
```

**機能要件**
- FR-O1: 企業アカウント発行（企業名＋初期管理者のメール招待）
- FR-O2: アシスタント登録と企業へのアサイン/解除（1人最大10社のバリデーション）
- FR-O3: 運営管理者は全企業のスレッドを閲覧できる（サポート・品質管理目的。閲覧は監査ログに記録）

## 3. 非機能要件

| # | 分類 | 要件 |
|---|---|---|
| NFR-1 | セキュリティ | 全認可判定（スレッド一覧・メッセージ・ファイルURL発行・WebSocket購読）をAPI層で強制。organization_id境界の自動テストを必須にする |
| NFR-2 | セキュリティ | 通信はTLSのみ。添付ファイルは非公開バケット＋署名付きURL（有効期限10分） |
| NFR-3 | セキュリティ | 監査ログ: 権限変更・メンバー招待/無効化・運営の企業データ閲覧を記録し、削除不可のストレージに保管 |
| NFR-4 | 性能 | メッセージ送信→他クライアント表示まで1秒以内（p95）。スレッド初期表示は直近50件をページング |
| NFR-5 | 性能 | 同時接続: 初期は 100社 × 平均5接続 = 500 WebSocket接続を1〜2インスタンスで捌ける設計（Goなら余裕） |
| NFR-6 | 可用性 | 営業時間帯（平日9-18時）の稼働率99.5%目標。Cloud Run min-instances=1（コールドスタート回避） |
| NFR-7 | データ保護 | Cloud SQL自動バックアップ（日次・7日保持）＋ポイントインタイムリカバリ。GCSはバージョニング有効 |
| NFR-8 | 切断耐性 | WebSocket切断（Cloud Run 60分制限・ネットワーク断・スリープ復帰）から5秒以内に自動再接続し、欠落メッセージを差分取得 |
| NFR-9 | 対応環境 | ブラウザ: Chrome/Edge/Safari 最新2バージョン。画面幅1024px以上を主対象（モバイルはネイティブアプリで対応予定） |
| NFR-10 | 言語 | 日本語のみ（i18n基盤は入れない） |

## 4. 確定履歴

2026-07-13 に以下を確定:
- メッセージ削除は「このメッセージは削除されました」の痕跡を残す（FR-H5）
- 既読は「誰が読んだか」まで表示するSlack風（FR-H3b）
- ファイルは1件100MB上限・保存期間無制限（FR-F3）
- 依頼カードの更新はシステムメッセージとしてチャットに集約（FR-R4）
